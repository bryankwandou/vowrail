# Vowrail iteration audit

## Brutal assessment before this iteration

The first pass had a strong landing page and credible domain language, but it was still a presentation prototype. The bind checklist mutated browser state, the claims review never called the drafting endpoint, the override control was decorative, and the renewal board could not run its sweep. Calling that a complete MVP would have been dishonest.

## Strategic correction

Vowrail should not attempt to replace established agency management systems. Those platforms already own broad policy, CRM, accounting, document, and servicing workflows. Vowrail wins as a focused control plane that can sit across an AMS: policy-type requirement graphs, hard bind gates, permanently distinct exceptions, grounded claims drafting, and independently scheduled renewal escalation.

## Defensible wedge

1. **Evidence graph:** agency-specific requirements linked to carrier and policy type.
2. **Decision gate:** a server and database transition that refuses an unsafe bind.
3. **Exception governance:** role-limited, justified overrides measured as risk signals.
4. **Outcome loop:** renewal and claims outcomes expose recurring operational failure patterns.
5. **Portable proof:** cryptographic evidence fingerprints can be anchored externally without exposing client data.

## Remaining launch risks

- The local MVP store proves behavior but production must use the supplied Supabase migration and authenticated agency tenancy.
- A real pilot must verify that configured requirements match carrier underwriting practice and agency procedure.
- Solana anchoring must contain only hashes and non-sensitive metadata; no client statement or policy PII belongs on-chain.
- Claims drafts need a fixed regression corpus reviewed by licensed agency staff before broad use.

## Validation target

The product is ready for a design-partner pilot when a principal can configure a policy type, an agent can fulfill evidence, an unsafe bind is rejected by both API and database, a justified override is permanently distinct, a claim draft cannot bypass staff review, and the renewal sweep produces no duplicate outreach interval.
